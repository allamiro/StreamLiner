Features

Built like modern log pipelines (think Vector, Fluent Bit, Logstash) but with an intentionally minimal flow model and deterministic normalization to common schemas (ECS/CEE/CLS).

Inputs

Syslog UDP/TCP

Parse RFC3164/5424; auto-detect timestamp formats and hostnames.

portsframingoctet-counting

File Tail

Reliable tail with inode tracking and rotation safety.

multilinebackpressure

HTTP

Ingest JSON/NDJSON via signed or basic-auth endpoints.

HMACCORS

Kafka

Consume/produce with exactly-once semantics (idempotent producer).

SASLTLS

Parsers & Normalization

Regex & Delimited

Capture groups and CSV/TSV splitting with type coercion and timezones.

JSON / CEF / LEEF

Robust parsing with escape handling and safe fallback to map fields.

Schema Maps

Preset mappers for ECS, CEE/CLS; author your own YAML transforms.

Enrichment

GeoIP/ASN, threat-intel tags, dedup, and lookup tables (CIDR, domains).

Outputs

OpenSearch / Elasticsearch

Bulk indexing with ILM-style rollover and retry with dead-letter queue.

S3 / MinIO

Write Parquet/NDJSON with partitioning by time/source for lakehouse use.

Kafka

Produce normalized topics per schema and route by severity or source.

Filesystem

Local dev-friendly output for quick inspection and test loops.

Operations

Observability

/metrics endpoint for Prometheus; structured logs with per-flow stats.

flow.processed, flow.dropped, output.retries

Resilience

Backpressure-aware queues, disk buffers, and exactly-once delivery where supported.

Security

mTLS, TLS pinning, API tokens, and field-level redaction/pseudonymization.

Footprint

Single binary or minimal container; runs well on edge nodes and lab VMs.

Feature parity with similar tools

Conceptually similar to Vector, Fluent Bit, and Logstash. StreamLiner focuses on deterministic normalization and a tiny, readable config for SIEM-centric workflows.
